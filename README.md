## Getting Started with Your Project

The information below will help point you in the direction for the information needed to complete the Student Directed Project. Do not modify the below.

1. Review the steps as found in the [Assignment README file](assignment/README.md)
1. Each week you need to show progress by making commits in GitHub. Drafts will be submitted to gauge your progress on programming.
1. Make sure all updates to files ending in `.md` use proper [Markdown Format](https://docs.github.com/en/get-started/writing-on-github/getting-started-with-writing-and-formatting-on-github/basic-writing-and-formatting-syntax).
1. Review the [Instructor Grading Rubric](rubric/INSTRUCTOR.md)
1. Update your [Student Grading Rubric](rubric/STUDENT.md) by Draft 1. The rubric must be finalized and approved by the instructor before the submission.
1. When a Pull Request is made by the instructor, merge the changes and handle conflicts.
    1. Merge the changes made by the instructor in the [Pull Request for the Student Rubric file](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/incorporating-changes-from-a-pull-request/merging-a-pull-request).
    1. After merging, make sure you pull the changes down into your local VS Code environment and handle any [Merge Conflicts](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/addressing-merge-conflicts/resolving-a-merge-conflict-using-the-command-line) that occur.
