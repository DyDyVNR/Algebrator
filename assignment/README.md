# Student Directed Project: Task List

Below is the task list for all of the items you need to complete as part of this assignment.

## Task 1. Software Engineering

Complete the [Software Engineering: Planning](1_PLANNING.md) section in the provided markdown file and submit for topic approval. Submit your repository to Canvas for project approval. Your instructor will complete a pull request with either topic approval or reject with requested changes.

## Task 2. Complete a Pull Request with Project Topic Approval

Merge your topic approval and start work on your project. By Draft 1 I should see changes to your source code on multiple days.

## Task 3. Draft 1 Check In

Complete the following items: 

* [Draft 1 Stand Up Meeting Questionnaire](../draft-1/README.md)
* [Software Engineering: Analysis Section](2_ANALYSIS.md)
* [Student Rubric](../rubric/STUDENT.md)

Submit your repository to Canvas for progress review and Student Rubric approval. If approved, complete a pull request otherwise refer to the Pull Request section for feedback on the required changes.

Continue work programming your software. By Draft 2, I should see additional changes to your source code on multiple days.

## Task 4. Draft 2 Check In

Complete the following items: 

* [Draft 2 Stand Up Meeting Questionnaire](../draft-2/README.md)
* [Software Engineering: Test Plan](3_TESTS.md)
* [Student Rubric](../rubric/STUDENT.md)

Submit your repository to Canvas for progress review. If your rubric was not approved during Draft 1 and is approved in this draft, complete a pull request otherwise refer to the Pull Request section for feedback on the required changes. This is your last chance to make changes to the Student Rubric. **Points will not be awarded during grading unless the instructor has completed the rubric approval process**. 

Continue work programming your software. By the final submission, I should see additional changes to your source code on multiple days.

## Task 5. Record Your Demonstration & Write Instructions

As a part of turning in your project, create a video showing the program working using the [Test Plan](4_TESTS.md) you wrote as a guide. You can use tools such as [Screencastify](https://www.screencastify.com/), [ScreenPal](https://screenpal.com/), [Zoom](https://zoom.us/), or more. Upload the video to YouTube and make sure it is either Unlisted or Public so the instructor can watch it.

Update the Implement section with the video link. Also, make sure to include any install instructions as well as how to execute the application.

* [Software Engineering: Video Section](4_IMPLEMENT.md)

## Task 6. Make Final Adjustments to Your Software Engineering

Make any final adjustments to the following items: 

* [Software Engineering: Planning](1_PLANNING.md)
* [Software Engineering: Analysis](2_ANALYSIS.md)
* [Software Engineering: Test Plan](3_TESTS.md)

## Task 7. Write Up Your Retrospective

Consider how the program went and complete a mini retrospective:

* [Retrospective](5_RETROSPECTIVE.md)

## Task 8. Submit Your Final Project

Congratulations! All you need to do now is go into Canvas and submit your project GitHub URL in the **Project Option 1: Completed Program** assignment.
